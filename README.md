# RUNBIRD株式会社ランバード 福祉事業ナレッジベース

Cursor AI と連携して、福祉事業の返信・法令対応を自動化するためのナレッジベースです。

## フォルダ構成

```
ナレッジ/
├── 返信例/          ← ベテランの返信パターン（AIが参照して返信文を自動生成）
│   ├── 利用者家族への連絡.md
│   ├── 行政への報告.md
│   ├── 他施設との連携.md
│   └── 苦情対応.md
├── 法令/            ← 法令・制度情報（県ごとに管理）
│   ├── 共通ルール.md
│   ├── 埼玉県.md
│   └── 触れてはいけないライン.md
├── 事業所/          ← 事業所固有の情報（手動メンテ）
│   └── Royal.md
└── Chatwork履歴/    ← OWL福祉事業 ChatworkからAPI取得（2024/4以降、Q&Aボット参照用）
    ├── _index.md       ← 142事業所の一覧
    ├── _corpus.md      ← 全文コーパス（5.7MB）
    ├── _summary.json   ← 取得結果サマリー
    ├── 事業所別/       ← 142事業所×時系列メッセージ
    └── raw/            ← 生JSONデータ
```

**Chatwork履歴の取得元:** OWL福祉事業 管理本部アカウント（owl-kanri@runbird.net）
**取得スコープ:** 解約系ルームを除外した172ルームのうち、2024/4以降にメッセージがある142ルーム計11,365件
**制約:** Chatwork APIは1ルーム最新100件上限のため、活発な事業所は古い履歴が欠落している可能性あり

## 使い方

1. **Cursor** を開いてこのフォルダを読み込む
2. AIに質問する（例：「利用者家族から○○の問い合わせが来た。返信して」）
3. AIがナレッジを自動で参照して返信文を生成
4. 内容を確認して送信

## ナレッジの追加・編集

1. 該当するフォルダのファイルをCursorで編集
2. GitHubにプッシュ（変更履歴が自動で残ります）
3. 次回からAIが最新のナレッジを参照します

## ルール

- **フォルダ構成は変更しない**（AIが参照できなくなります）
- **既存ファイルの書き方に合わせる**（パターン → 返信例 → ポイントの形式）
- **法令情報を変更したら更新履歴に記録する**
- **不明な点は管理者に確認してから編集する**
