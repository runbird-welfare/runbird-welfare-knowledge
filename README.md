### 用途別の参照ファイル

| やりたいこと | 見るファイル |
|---|---|
| 事業所を特定したい | `ナレッジ/事業所/索引.md` |
| ある事業所の経緯を知りたい | `ナレッジ/事業所/履歴/{コード}_*.md` |
| 加算について教えてほしい | `ナレッジ/加算/README.md` → 該当カテゴリ |
| 返信文を作りたい | `ナレッジ/返信例/{テーマ}.md`（実例: `_抽出_*.md`） |
| 法令を確認したい | `ナレッジ/法令/{県名}.md` |

### Chatwork履歴の更新方法

斗（hishaku）リポジトリ側のスクリプトで再取得します（OWLのAPIトークンを参照するため）。

```bash
cd ~/Documents/斗（hishaku）
python3 tools/fetch_owl_chatwork.py
```

実行後、`事業所/履歴/` `加算/` `返信例/_抽出_*.md` `Chatwork履歴/` がすべて再生成されます。`raw/*.json` は `.gitignore` でgit管理外。

## 使い方

1. **Cursor** を開いてこのフォルダを読み込む
2. AIに質問する（例：「利用者家族から○○の問い合わせが来た。返信して」）
3. AIがナレッジを自動で参照して返信文を生成
4. 内容を確認して送信

## ナレッジの追加・編集

1. 該当するフォルダのファイルをCursorで編集
2. GitHubにプッシュ（変更履歴が自動で残ります）
3. 次回からAIが最新のナレッジを参照します

## ルール

- **フォルダ構成は変更しない**（AIが参照できなくなります）
- **既存ファイルの書き方に合わせる**（パターン → 返信例 → ポイントの形式）
- **法令情報を変更したら更新履歴に記録する**
- **不明な点は管理者に確認してから編集する**
- **`_抽出_*.md` `事業所/履歴/*.md` `加算/*.md` `Chatwork履歴/*` は自動生成ファイル** — 直接編集しても次回スクリプト実行時に上書きされます。手で修正したい場合はキュレーション済みファイル（`Royal.md` `利用者家族への連絡.md` 等）に転記してください。
